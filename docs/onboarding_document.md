# Onboarding Guide

## 1. Request Access to VDI
To begin your onboarding process, please request access to the Virtual Desktop Infrastructure (VDI). This will enable you to work within the secure environment required for your role.

**Steps:**
1. Submit a VDI access request through the internal IT portal.
2. Await approval notification from IT.

---

## 2. Request Access to Required AD Groups
Based on your use case, request access to the following Active Directory (AD) groups:

- **grup1**
- **grup1**

*Note: Ensure that your manager confirms the appropriate groups for your role before submitting your request.*

---

## 3. Access to GitHub
Access to the organization's GitHub repositories is required for development activities.

**Steps:**
1. Submit a GitHub access request via the access management system.
2. Specify your GitHub username and team/project.
3. Wait for confirmation from the DevOps or IT admin team.

---

## 4. Workspace Information
You will be working within the following workspaces depending on your environment needs:

- **Workspace - Dev**
- **Workspace - Prod**

Ensure you have the correct permissions for each workspace based on your role and responsibilities.

---

If you need assistance during onboarding, please reach out to your manager or the IT support team.
